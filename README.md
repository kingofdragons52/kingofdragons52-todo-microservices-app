# 📋 Task Tracker (Trello Style)

Многопользовательское SPA-приложение (Single Page Application) для управления задачами, построенное на **микросервисной архитектуре**. Проект демонстрирует навыки работы с распределенными системами, брокерами сообщений и контейнеризацией.

---

## Основной функционал

### 🔐 Работа с пользователями
* **Регистрация и Авторизация:** Безопасный вход с использованием **JWT (JSON Web Token)**.
* **Автоматический вход:** После регистрации пользователь сразу получает доступ к системе.
* **Профиль:** Отображение текущего авторизованного пользователя.

### 📝 Управление задачами
* **Dashboard:** Разделение задач на "В работе" и "Выполнено".
* **Live-редактирование:** Автосохранение заголовка и описания при потере фокуса (без кнопки "Сохранить").
* **Управление статусом:** Возможность мгновенно пометить задачу как выполненную или вернуть в работу.
* **Удаление:** Полное удаление задачи из системы.

### 📧 Система уведомлений
* **Welcome Email:** Моментальная отправка приветственного письма после успешной регистрации.
* **Daily Report:** Ежедневный отчет в полночь со статистикой: сколько задач выполнено за сутки и сколько осталось в очереди.

---

## 🏗 Архитектура и сервисы

Приложение разделено на независимые модули, взаимодействующие через REST и брокер сообщений:

1.  **`task-tracker-backend`**: Центральный API. Хранит данные в PostgreSQL, управляет бизнес-логикой и генерирует события для Kafka.
2.  **`task-tracker-frontend`**: Статический клиент на HTML/jQuery, работающий внутри Nginx.
3.  **`task-tracker-scheduler`**: Сервис-планировщик. Раз в сутки опрашивает бэкенд и формирует задания на рассылку отчетов.
4.  **`task-tracker-email-sender`**: Почтовый агент. Слушает Kafka-топик и отправляет реальные письма через SMTP.

## 💻 Запуск проекта

### Требования
*   Установленный **Docker** и **Docker Compose**.
*   **Maven** (для локальной сборки артефактов).

### Шаги для запуска:

1. **Сборка всех модулей (JAR):**
   ```bash
   mvn clean package -DskipTests

2.  Запуск контейнеров:

    docker-compose up --build

3.  Доступ к приложению:

      - Frontend: http://localhost
      - API (Backend): http://localhost:8080
      - Почтовый ящик (MailHog): http://localhost:8025

📈 Схема взаимодействия (Data Flow)

1.  Пользователь сохраняет задачу → Frontend шлет AJAX (PUT) на Backend.
2.  Backend обновляет БД и (если нужно) шлет событие в Kafka.
3.  Email-Sender вычитывает событие из Kafka и шлет письмо через MailHog.
4.  Scheduler ночью стучится в Backend, собирает данные и шлет отчеты в Kafka.

⚙️ Переменные окружения (Secrets)

Основные настройки (БД, Кафка) прописаны в docker-compose.yml. Для смены SMTP на
реальный (например, Mailjet), обновите переменные в сервисе email-sender.


## 🔍 Тестирование API

Большинство запросов требуют передачи JWT-токена в заголовке `Authorization`. Токен выдается автоматически при регистрации или логине.

### 1. Работа с пользователями

#### **Регистрация**
`POST /user`
*   **Тело запроса:**
    ```json
    {
      "email": "user@example.com",
      "password": "yourpassword"
    }
    ```
*   **Особенности:** В случае успеха возвращает JWT-токен в HTTP-заголовке `Authorization` (Bearer token).

#### **Авторизация (Логин)**
`POST /auth/login`
*   **Тело запроса:** Аналогично регистрации.
*   **Особенности:** Возвращает JWT-токен в заголовке `Authorization`.

#### **Получение текущего пользователя**
`GET /user`
*   **Заголовок:** `Authorization: Bearer <ваш_токен>`
*   **Ответ:** `{"id": 1, "email": "user@example.com"}`

---

### 2. Работа с задачами

#### **Получение списка всех задач**
`GET /tasks`
*   **Заголовок:** `Authorization: Bearer <ваш_токен>`

#### **Создание новой задачи**
`POST /tasks`
*   **Заголовок:** `Authorization: Bearer <ваш_токен>`
*   **Тело запроса:**
    ```json
    {
      "title": "Купить продукты",
      "description": "Молоко, хлеб, сыр",
      "isDone": false
    }
    ```

#### **Обновление задачи (включая статус "выполнено")**
`PUT /tasks/{id}`
*   **Заголовок:** `Authorization: Bearer <ваш_токен>`
*   **Тело запроса:**
    ```json
    {
      "title": "Купить продукты",
      "description": "Молоко, хлеб, сыр, яблоки",
      "isDone": true
    }
    ```

#### **Удаление задачи**
`DELETE /tasks/{id}`
*   **Заголовок:** `Authorization: Bearer <ваш_токен>`

---

### 3. Служебные эндпоинты

#### **Получение данных для ежедневных отчетов**
`GET /tasks/internal/daily-reports`
*   **Особенности:** Используется сервисом `scheduler`.
*   **Заголовок:** `X-Internal-Secret: my-super-shared-secret-key-2026`
